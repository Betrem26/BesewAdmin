THE ISSUE
================================================================================

The PUT endpoint was rejecting all requests with "Input validation failed" error.

ROOT CAUSE IDENTIFIED:
The API documentation shows that the PUT endpoint does NOT accept "menuId" in
the request body. The menuId is ONLY in the URL path parameter.

API Specification:
PUT /api/v1/menu-config/{menuId}

Request Body (does NOT include menuId):
{
  "label": "string",
  "path": "string",
  "icon": "string",
  "description": "string",
  "badge": "string",
  "isActive": true,
  "allowedUserTypes": ["string"],
  "allowedWorkerTypes": ["string"],
  "minTrustScore": 0,
  "allowedSubscriptionTiers": ["string"],
  "order": 0,
  "parentMenuId": "string"
}

================================================================================
THE FIX
================================================================================

Modified the handleSave() and handleToggleActive() functions to send ONLY the
fields that the API accepts (excluding menuId from the request body).

BEFORE:
const updateData = {
  menuId: editData.menuId,  // ❌ NOT ACCEPTED BY API
  label: editData.label,
  path: editData.path,
  ...
};

AFTER:
const updateData = {
  // menuId is NOT included in the body
  label: editData.label,
  path: editData.path,
  icon: editData.icon,
  description: editData.description,
  badge: editData.badge,
  isActive: editData.isActive,
  minTrustScore: editData.minTrustScore || 0,
  allowedUserTypes: editData.allowedUserTypes || [],
  allowedWorkerTypes: editData.allowedWorkerTypes || [],
  allowedSubscriptionTiers: editData.allowedSubscriptionTiers || [],
  order: editData.order,
  parentMenuId: editData.parentMenuId
};

The menuId is still passed to the API via the URL:
dispatch(updateMenuConfig({ menuId: editData.menuId, data: updateData }))
