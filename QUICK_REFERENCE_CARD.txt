WHAT WAS FIXED:
  1. Counts showing 0 (Total, Pending, Verified, Rejected)
  2. Verified button not updating counts
  3. Counts cleared after page refresh

ROOT CAUSES:
  • Stats endpoint failing silently with no fallback
  • Stats only calculated conditionally (if prev.total === 0)
  • No backend sync after verification
  • No fallback calculation when stats endpoint fails

SOLUTION:
  • Added fallback calculation in loadStats()
  • Always calculate stats from company list in loadCompanies()
  • Recalculate stats after verification/deletion
  • Sync with backend for persistence

FILE MODIFIED:
  src/pages/dashboard/CompanyManagement.tsx

FUNCTIONS UPDATED:
  1. loadStats() - Added fallback calculation
  2. loadCompanies() - Always calculate stats
  3. handleVerificationSuccess() - Recalculate & sync
  4. handleDeleteCompany() - Recalculate stats

HOW IT WORKS:
  Page Load
    ↓
  Load Companies → Calculate Stats from List
    ↓
  Load Stats from Backend (or use fallback)
    ↓
  Display Counts (always accurate)
    ↓
  User Verifies Company
    ↓
  Recalculate Stats → Sync with Backend
    ↓
  Counts Update Correctly
    ↓
  User Refreshes Page
    ↓
  Counts Persist (from backend or fallback)
