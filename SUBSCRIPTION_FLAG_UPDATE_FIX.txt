================================================================================
SUBSCRIPTION MANAGEMENT - FLAG UPDATE FIX
================================================================================

ISSUE:
Subscription flag updates (isPopular, isRecommended) were failing with 400 
"Input validation failed" error.

ROOT CAUSE:
The PUT request was not sending all required fields defined in the backend 
schema. The backend validation requires ALL fields to be present, even if 
only updating one flag.

================================================================================
SOLUTION IMPLEMENTED
================================================================================

Updated the handleFlagUpdate() function to:

1. SANITIZE DATA
   - Remove MongoDB-specific fields: _id, __v, createdAt, updatedAt
   - These fields are not accepted by the backend PUT endpoint

2. INCLUDE ALL REQUIRED FIELDS
   - type: Must match the existing plan type
   - period: Must match the existing plan period
   - price: Sent as Number (not string)
   - durationInDays: Sent as Number (not string)
   - features: Sent as Array (not null/undefined)
   - description: Sent as string (empty string if not provided)
   - isActive: Current status (boolean)
   - isPopular: Updated value (boolean)
   - isRecommended: Updated value (boolean)
   - sortOrder: Current sort order (number)
   - maxFeatures: Only included if it has a value

3. VALIDATION SYNC
   - Payload now matches the Swagger definition exactly
   - All field types are correct (Number, String, Array, Boolean)
   - No extra fields that could cause validation errors

4. DEBUG LOGGING
   - Added console.log('Sending Payload:', payload) before API call
   - Shows exactly what is being sent to the server
   - Helps diagnose future validation issues

================================================================================
CODE CHANGES
================================================================================

FILE: src/pages/dashboard/SubscriptionManagement.tsx

Function: handleFlagUpdate()

BEFORE:
```typescript
await subscriptionApi.update(plan._id, buildUpdatePayload(plan, { [field]: value }));
```

AFTER:
```typescript
const payload = {
  type: plan.type,
  period: plan.period,
  price: Number(plan.price),
  durationInDays: Number(plan.durationInDays),
  features: Array.isArray(plan.features) ? plan.features : [],
  description: plan.description || '',
  isActive: plan.isActive ?? true,
  isPopular: field === 'isPopular' ? value : (plan.isPopular ?? false),
  isRecommended: field === 'isRecommended' ? value : (plan.isRecommended ?? false),
  sortOrder: plan.sortOrder ?? 0,
};

if (plan.maxFeatures !== undefined && plan.maxFeatures !== null) {
  payload.maxFeatures = Number(plan.maxFeatures);
}

console.log('Sending Payload:', payload);
await subscriptionApi.update(plan._id, payload);
```
