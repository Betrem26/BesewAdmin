ISSUES FIXED:

1. ✅ STYLED-COMPONENTS WARNINGS
   Problem: Props like 'active', 'isChild', 'level', 'variant' were being passed
            to DOM elements, causing React warnings
   Solution: Converted to transient props using '$' prefix
   
   Files Modified:
   - src/pages/settings/Settings.tsx
     * Tab component: active → $active
   
   - src/pages/settings/MenuManagement.tsx
     * Button component: variant → $variant
     * Alert component: type → $type
     * Tr component: isChild → $isChild
     * MenuLabel component: level → $level
   
   Updated JSX:
   - <Tab $active={activeTab === 'menu'} />
   - <Button $variant="danger" />
   - <Alert $type="error" />
   - <Tr $isChild={isChild} />
   - <MenuLabel $level={level} />

2. ✅ ERROR OBJECT RENDERING
   Problem: API error response object was being rendered directly in JSX,
            causing "Objects are not valid as a React child" error
   Solution: Added type checking and safe property access
   
   Before:
   <span>{error}</span>
   
   After:
   <span>{typeof error === 'string' ? error : (error as any)?.message || 'An error occurred'}</span>

3. ✅ TYPESCRIPT ERRORS
   Problem: Property 'message' does not exist on type 'never'
   Solution: Added type assertion (error as any) for safe access
